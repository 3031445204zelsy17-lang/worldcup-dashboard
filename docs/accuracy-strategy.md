# 准确性保障策略（五层把控）

> 配合 [[project-context.md]] 阅读。这里放准确性保障的详细方法。

---

## 核心原则

**科学不在于预测准不准，而在于有没有诚实告诉用户"我不知道的部分有多大"。**

---

## 第一层：数据质量

| 环节 | 做法 |
|------|------|
| **数据源选择** | API-Football + football-data.org 双源交叉验证，不一致标记存疑 |
| **历史数据量** | 4 年国际比赛（2022-2026），~15,000 场 |
| **数据权重衰减** | Dixon-Coles 时间衰减函数 ρ(t)，近期权重更高 |
| **赛事等级区分** | 世界杯决赛 ≠ 友谊赛，不同赛事不同权重 |

---

## 第二层：模型准确

### 三个层次的模型
```
Level 1: Poisson 基础模型          → 简单但低分比赛准确率差
Level 2: Dixon-Coles 改进模型      → 修正低分偏差 + 时间衰减
Level 3: 动态调整（实时胜率）      → 比赛中持续修正
```

### 验证方法（最关键）

| 方法 | 怎么做 | 判定标准 |
|------|--------|---------|
| **历史回测** | 用 2018/2022 WC 数据 backtest，预测 vs 实际 | 基准 60% 准确率 |
| **Brier Score** | 预测概率 vs 实际结果的均方误差 | 越低越好（0=完美） |
| **Calibration Plot** | "预测 70% 胜"的比赛，实际是否 ~70% 胜 | 曲线贴近对角线 = 可靠 |
| **Log Loss** | 评估概率预测质量，惩罚过度自信 | 和博彩赔率反推对比 |

---

## 第三层：实时修正

赛中根据 game state 持续调整：
```
赛前基础概率 (Poisson/Dixon-Coles)
    ↓ + 当前比分
    ↓ + 已过时间（剩余分钟数）
    ↓ + 红牌/受伤
    ↓ + xG 数据（如果有）
    ↓
实时胜率
```

核心方法：**Poisson 剩余时间模拟**
```
第 70 分钟，比分 1:0，对方红牌
→ 剩余 20 分钟，重算双方进球期望
→ 跑 Poisson 算剩余时间进球分布
→ 加总 → 当前胜/平/负概率
```

## 第四层：对标

| 对标对象 | 做法 |
|---------|------|
| **博彩公司赔率** | 赔率反推隐含概率，和模型对比。Brier Score 接近赔率模型 = 很好 |
| **ESPN/FiveThirtyEight** | 对比公开预测 |
| **Elo Rating 基准** | 纯 Elo 排名预测作 baseline，模型必须优于它 |

---

## 第五层：透明度

| 做法 | 说明 |
|------|------|
| **显示置信区间** | "巴西 45% ± 8% 胜"，不只给点估计 |
| **展示模型依据** | "基于近 20 场 + Elo 差 120 分" |
| **历史准确率** | 仪表盘放"过去 N 场预测准确率"指标 |
| **免责声明** | "基于统计模型，仅供参考" |

## 处理"不可预测"因素

### 真正随机的（任何模型都预测不了）
- 赛中突发伤病
- 裁判争议判罚
- 天气突变
- 心理崩溃/爆发
- 黑天鹅事件

**这些不是模型问题，是世界本质。** 博彩公司也一样。

### 可部分处理的（赛前有信号）
| 因素 | 处理方式 |
|------|---------|
| 赛前伤病名单 | 降低球员/球队进攻期望 |
| 主客场 | 固定参数 |
| 海拔/气候 | 环境修正系数 |
| 赛程密集度 | 体能下降 → 进球期望降低 |
| 出线形势 | 已出线队可能轮换（人工判断） |

### 贝叶斯方法处理动态信息
```
赛前：巴西 60% 胜（先验）
    ↓ 看到首发（内马尔没上）
更新：巴西 48% 胜（后验）
    ↓ 看到下雨（传控受影响）
再更新：巴西 42% 胜
```
footBayes R 包就是这个思路。

### 模型集成（多模型投票）
```
Poisson       → 65%
Dixon-Coles   → 58%
Elo           → 60%
赔率反推       → 55%
加权平均       → 59.5% ± 4%
```
模型间差异大 → 不确定性高 → 展示更宽置信区间。

---

## 准确性优先级排序

```
最重要 ──→ 次要
─────────────────────────────────
1. Brier Score 回测验证    ← 不做等于盲飞
2. Dixon-Coles + 时间衰减   ← 比 Poisson 明显更准
3. 双数据源交叉验证         ← 防脏数据
4. 博彩赔率对标             ← 知道自己在什么水平
5. 实时 game state 修正     ← 比赛中准确性跃升
6. xG 数据整合              ← 锦上添花
```

| 方法 | 预期准确率 |

| 纯 Elo | ~58-60% |
