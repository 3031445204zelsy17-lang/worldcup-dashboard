### 区域说明
| 区 | 数据来源 | 交互 |
|----|---------|------|
| 状态条 | `matches`（赛期/倒计时）、`tournament_probs`（更新计数） | 无 |
| 夺冠榜 | `tournament_probs`（win_prob），日变化 = 当次重算 vs 上次 | 点行 → 球队详情；`[夺冠\|出线]`切换排序键；`ⓘ` → 方法论 |
| 今日比赛 | `matches`（今日 kickoff）+ `predictions`（赛前 prob） | 点卡片 → 比赛详情 |
| 今日变动 | `tournament_probs` 两次快照 diff，按 \|Δ\| 排序 Top N | 点行 → 球队详情 |

### 夺冠榜排序与视图切换
- **默认排序**：夺冠概率降序（话题性最强，首页第一钩子）
- **视图切换**（tab，随赛程阶段自动调整可选维度）：
  - 小组赛阶段：`[夺冠概率]` `[小组出线概率]`
  - 淘汰赛阶段：`[夺冠概率]` `[当前轮次晋级概率]`（出线已无意义，自动替换）
- **每行始终显示**：国旗 · 队名 · 当前视图概率（大）· 夺冠概率（小，灰）· 日变化 `↑↓`
- **依据**：阶段不同用户关心的概率不同——小组赛盯出线生死，淘汰赛盯能走多远。默认统一用夺冠排序避免认知割裂，但提供阶段相关的第二视图。

```
│   夺冠     ██░░░░░░░░░░░░░░░░░  18%  ⓘ          │
├──────────────────────────────────────────────────┤

│  📅 巴西赛程                                      │

├──────────────────────────────────────────────────┤
│  🔍 为什么是 18%？  [展开]                         │  透明度
│   • 实力 #3 · 近10场 7胜2平1负                    │
│   • 核心前锋在阵 ✓ · 无关键伤病                    │
└──────────────────────────────────────────────────┘
```

### 区域说明
| 区 | 数据来源 |
|----|---------|
| 实力头 | `teams`（elo, group）、实时积分 |
| 晋级路径 | `tournament_probs` 按 round 展开（advancement_prob 逐轮 + win_prob）|
| 对阵树 | `matches`（淘汰赛对阵）+ 概率标注 |
| 赛程 | `matches`（该队 involved）+ `predictions` |
| 为什么展开 | `teams`（elo）、近期战绩、`injuries`（关键球员在阵）|

**核心设计**：阶梯式晋级概率是这个页最独特的呈现——竞品通常只给"夺冠概率"一个数字，这里给整条路径，每多走一轮概率递减，用户直观感受"离夺冠还隔几道坎"。

### 区域说明
| 区 | 数据来源 | 阶段 |
|----|---------|------|
| 胜平负 | `predictions`（赛前 prob）+ `teams`（elo 对比）| A |
| 实时曲线 | A：赛后回放 `predictions` 序列；B：赛中 WebSocket 实时 + `events` 转折标注 | **A 留位 / B 填充** |

| 比分分布 | Poisson 模型输出 | A |

**关键**：实时曲线区在 A 阶段先留位，B 阶段往里填实时 + 转折点标注（"72′ 红牌，胜率 61%→38%"）。**A 不返工，B 往留位填东西。**

## 贯穿三条设计原则

1. **信息层级递进**：每页都是「核心数字 → 驱动因素 → 方法论」三段，想看多深点多深
2. **不确定性可视化**：概率条旁挂 `ⓘ`（置信区间/阴影带），不假装精确——竞品都没做
3. **晨起回看线**：A 阶段"今日变动"，B 阶段无缝叠加"实时转折点回放"，同一条信息线
