## 待设计项（按优先级）

### 1️⃣ 产品范围（最优先）
- [ ] MVP 精确功能清单（到底做哪几个页面/功能）
- [ ] 明确"不做什么"（砍掉的边界）
- [ ] 单场实时胜率 vs 锦标赛概率，MVP 先做哪个？
- [ ] 是否包含历史比赛查询，还是只看当前/未来？

### 2️⃣ 用户旅程
- [ ] 目标用户画像（自己用？给同学看？发网上？）
- [ ] 比赛日 vs 非比赛日，体验差异
- [ ] 用户第一次打开看到什么 → 点哪里 → 看到什么
- [ ] 核心使用场景（看预测？看实时？看晋级？）

### 3️⃣ 核心页面 & 线框图 ✅
- [x] 总共几个页面？→ **4 个**（总览 / 球队详情 / 比赛详情 / 方法论），见 [[wireframes.md]]
- [x] 首页布局（线框图）→ 总览页即首页，见 [[wireframes.md]]
- [x] 单场比赛详情页布局 → 见 [[wireframes.md]]
- [x] 锦标赛总览页布局 → 见 [[wireframes.md]]
- [x] 导航结构 → 总览为枢纽，见 [[wireframes.md]]

### 4️⃣ 信息架构
- [ ] 每个页面展示哪些数据
- [ ] 数据优先级（核心数字 vs 辅助信息）
- [ ] 交互层级（点击展开？弹窗？新页面？）

### 5️⃣ 视觉风格
- [ ] 整体调性（专业数据感？活泼？极简？）
- [ ] 配色方案（参考竞品？自定义品牌色？）
- [ ] 图表风格（Recharts 默认？定制？）
- [ ] 是否需要 logo / 品牌名

### 6️⃣ 功能优先级（MoSCoW）
- [ ] Must have（MVP 必须有）
- [ ] Should have（MVP 后很快加）
- [ ] Could have（Phase 2-3）
- [ ] Won't have（明确不做）

### 7️⃣ 成功标准
- [ ] 怎么算"做成了"？（准确率？体验？完成度？）
- [ ] 上线目标日期（WC 决赛 7/19 前？）
- [ ] 验收标准（能跑通哪些场景）

## 设计方法论建议

每个待设计项可以用以下流程：
1. **我先给 2-3 个方案**（带 pros/cons）
2. **你选一个或提修改**
3. **定稿写进文档**

不要一上来就完美，迭代着来。
